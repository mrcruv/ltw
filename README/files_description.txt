ltw-main

    /css
        /style.css: foglio di stile utilizzato per la grafica rincorrente nelle pagine principali.

    /includes
        /close_connection.php: file php che effettua la chiusura della connessione con il server prendendo le informazioni di configurazione includendo il file config.php.
        /config.php: file php contenente le informazioni di configurazione del server e del database.
        /error:php: file php contenente il modal visualizzato in caso di errore lato server attraverso la variabile globale $_GET.
        /footer.php: file php contentente il footer visualizzato in tutte le principali pagine del sito dopo aver effettuato il login.
        /header.php: file php contentente il header visualizzato in tutte le principali pagine del sito dopo aver effettuato il login. 
        /info.php: file php contentente le informazioni del sito visualizzate nel footer. 
        /lengths.php: file php contentente le dimensioni minime e massime ammissibili per le validazioni lato server delle principali operazioni di login, register, add, update e delete. 
        /message.php: file php contenente il modal visualizzato in caso di successo di un'operazione (login, update, ecc..) attraverso la variabile globale $_GET.
        /open_connection.php: file php che effettua l'apertura della connessione con il server prendendo le informazioni di configurazione dal file config.php.
        /regex.php: file php contentente le espressioni regolari per le validazioni lato server delle principali operazioni di login, register, add, update e delete. 
        /session.php: file php che effettua avviare la sessione con un controllo in caso di connessione già avviata. 

    /scripts
        /accept_reject.php: file php che effettua l'accettazione o il rifiuto di una assegnazione pendente. Dopo i controlli opporturni viene aggiornata la data di accettazione, nel caso l'esperto accetta, o la data di rifiuto nel caso l'esperto rifiuta.
        /add_availability.php: file php che effettua l'aggiunta di una assegnazione da parte di un ente. Dopo i controlli opportuni viene aggiunta l'assegnazione dell'ente con il processo e l'esperto scelti.
        /add_competence.php: file php che effettua l'aggiunta da parte di un esperto di una competenza. Viene effettuata la validazione lato server includendo i file regex.php e lengths.php.
        /add_form.js: file javascript che genera l'animazione a comparsa dei form di aggiunta nelle pagine titoli.php, assegnazioni.php, competenze.php, processi.php.
        /add_process.php: file php che effettua l'aggiunta da parte di un ente di un processo. Viene effettuata la validazione lato server includendo i file regex.php e lengths.php.
        /add_title.php: file php che effettua l'aggiunta da parte di un esperto di un titolo di studio. Viene effettuata la validazione lato server includendo i file regex.php e lengths.php.
        /delete_competence.php: file php che effettua l'eliminazione da parte di un esperto di una competenza.
        /delete_title.php: file php che effettua l'eliminazione da parte di un esperto di un titolo di studio.
        /entity_stats.php: file php che effettua il calcolo delle statistiche di un ente. Vengono visualizzati il numero di: processi, assegnazioni (divise in pendenti, accettate, rifiutate).
        /error.js: file javascript che mostra il modal di errore.
        /expert_stats.php: file php che effettua il calcolo delle statistiche di un esperto. Vengono visualizzati il numero di: titoli di studio, competenze, assegnazioni (divise in pendenti, accettate, rifiutate). 
        /form_switch.js: file javascript che genera l'animazione del form di login e registrazione di index.php.
        /login.php: file php che effettua il login e setta la variabile $_SESSION con username e tipo di utente.
        /logout.php: file php che effettua il logout, l'unset della variabile $_SESSION e distrugge la sessione.
        /message.js: file javascript che mostra il modal di successo dell'operazione eseguita.
        /register_entity.php: file php che effettua la registrazione di un ente. Viene effettuata la validazione lato server includendo i file regex.php e lengths.php.
        /register_expert.php: file php che effettua la registrazione di un esperto. Viene effettuata la validazione lato server includendo i file regex.php e lengths.php.
        /register_slide.js: file javascript che genera l'animazione di scorrimento del form di registrazione nella pagina di index.php.
        /remove_website.php: file php che effettua l'eliminazione da parte di un utente del sito web.
        /show_availability.php: file php formato da due funzioni che memorizzano in un array tutte le assegnazioni di un ente (funzione ente) o di un esperto (funzione esperto) ritornando l'array stesso.
        /show_competence.php: file php formato da una funzione che memorizza in un array tutte le competenze di un esperto ritornando l'array stesso.
        /show_expert.php: file php formato da una funzione che memorizza in un array tutti gli esperti ritornado l'array stesso. Due funzioni che memorizzano in degli array i titoli di studio e le competenze di ogni esperto che vengono ritornati. Infine una quarta funzione che controlla se l'esperto esiste. 
        /show_process.php: file php formato da una funzione che memorizza in un array tutte i processi di un ente ritornando l'array stesso.
        /show_title.php: file php formato da una funzione che memorizza in un array tutte i titoli di studio di un esperto ritornando l'array stesso.
        /toggle_psw.js: file javascript per il toggle della visibilità della password.
        /update_cf.php: file php che effettua l'update del Codice Fiscale di un utente. Viene effettuata la validazione lato server includendo i file regex.php e lengths.php.
        /update_entity_name.php: file php che effettua l'update del nome dell'ente. Viene effettuata la validazione lato server includendo i file regex.php e lengths.php.
        /update_entity_type.php: file php che effettua l'update del tipo dell'ente. Viene effettuata la validazione lato server includendo i file regex.php e lengths.php.
        /update_info.js: file javascript 
        /update_password.php: file php che effettua l'update della password di un utente. Viene effettuata la validazione lato server includendo i file regex.php e lengths.php.
        /update_pec.php: file php che effettua l'update della PEC di un utente. Viene effettuata la validazione lato server includendo i file regex.php e lengths.php.
        /update_piva.php: file php che effettua l'update della Partita IVA di un utente. Viene effettuata la validazione lato server includendo i file regex.php e lengths.php.
        /update_website.php: file php che effettua l'update del sito web di un utente. Viene effettuata la validazione lato server includendo i file regex.php e lengths.php.
        /validate_add_competence.js: file javascript che effettua la validazione dell'aggiunta di una competenza attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni.
        /validate_add_process.js: file javascript che effettua la validazione dell'aggiunta di un processo attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni.
        /validate_add_title.js: file javascript che effettua la validazione dell'aggiunta di un titolo di studio attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni. 
        /validate_login.js: file javascript che effettua la validazione del login di un utente attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni. 
        /validate_register_entity.js: file javascript che effettua la validazione della registrazione attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni.
        /validate_register_expert.js: file javascript che effettua la validazione dell'aggiunta di una competenza attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni.
        /validate_update_cf.js: file javascript che effettua la validazione dell'update del Codice Fiscale di un utente attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni.
        /validate_update_entity_name.js: file javascript che effettua la validazione dell'update del nome di un ente attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni.
        /validate_update_entity_type.js: file javascript che effettua la validazione dell'update del tipo di un ente attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni.
        /validate_update_password.js: file javascript che effettua la validazione dell'update della password di un utente attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni.
        /validate_update_pec.js: file javascript che effettua la validazione dell'update della PEC di un utente attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni.
        /validate_update_piva.js: file javascript che effettua la validazione dell'update della Partita IVA di un utente attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni.
        /validate_update_website.js: file javascript che effettua la validazione dell'update del sito web di un utente attraverso il metodo validate() di JQuery. Viene effettuata la vaidazione lato client con esperessioni regolari e minime e massime dimensioni. 

    /assegnazioni.php: file php che visualizza tutte le assegnazioni che ha un utente (pendenti, accettate e rifiutate) e nel caso l'utente sia un ente viene visualizzato anche un form a comparsa per l'aggiunta di un'assegnazione.
    /competenze.php: file php che visualizza tutte le competenze che ha un utente esperto. Viene visualizzato anche un form a comparsa per l'aggiunta di una competenza.
    /esperti.php: file php che visualizza tutti gli esperti registrati al sito. In questa pagina un utente ente può visualizzare tutte le informazioni di un esperto (titoli di studio e competenze).
    /index.php: file php che visualizza i form di login e registrazione e una breve descrizione del sito.
    /me.php: file php che visualizza la dashboard dell'utente con tutte le informazioni, statistiche e form di cambio password.
    /processi.php: file php che visualizza tutti i processi che un ente ha creato. Viene visualizzato anche un form a comparsa per l'aggiunta di un processo.
    /titoli.php: file php che visualizza tutti i titoli di studio di un utente esperto. Viene visualizzato anche un form a comparsa per l'aggiunta di un titolo di studio.
